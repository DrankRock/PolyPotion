CHAPTER VI · SCULPT · JULY 20, 2026

The brushes feel like Blender. The round-trip doesn't yet.

The sculpt pass turned a five-brush toy into a fourteen-brush surface tool with dyntopo, masks, symmetry axes, redo and material preview. As a place to push clay it now holds its own. But a sculptor's mesh doesn't live in a vacuum — it arrives textured and rigged, and it has to leave the same way. That's where the tool still quietly breaks: saving a sculpt discards the character's UVs, materials and textures, and the one feature that would make browser-sculpting genuinely worth doing — baking the detail down to a normal map — isn't wired, even though the tool it needs is already in the suite.

The order is obvious. Fix the export first — a sculpt tool that destroys the character's textures on save is worse than no tool, because the damage is silent. Then wire the normal-map bake: it's the reason to sculpt in a browser at all, and BakeMaps already does the hard part. Everything after that is refinement — a spatial hash so dyntopo can run past 200k without stutter, a remesh option to clean the density dyntopo creates, pressure→radius, and the checkpoint/autosave that ties sculpt into the wider brew-history idea. None of it needs a server; all of it composes with tools already shipped.
